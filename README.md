# feiniao-private-core

飞鸟私有主线仓库。

## 三线分工

### 1. 官方参考线
- 站点：`https://feiniao.112book.com/`
- 用途：只跟官方更新，只做对照，不作为正式开发主线。

### 2. 基准站 / 私有母版
- 站点：`https://feiniaojizhun.112book.com/`
- 用途：仓库主线、私有母版、后续新资源站实例复制来源。
- 规则：只保留已经验证正确的内容，不做高风险测试。

### 3. 测试站
- 站点：`https://feiniaoceshi.112book.com/`
- 用途：测试修改、测试插件、测试模板、测试站群配置、测试 API 对接。
- 规则：先在测试站验证，确认没问题后，再同步到基准站和仓库。

## 当前协作规则

1. 测试站先改，基准站后收口。
2. 基准站只接收已经验证通过的修改。
3. 仓库内容以基准站为准，不以测试站为准。
4. `runtime/`、日志、缓存、临时文件不入仓库。
5. `vendor/` 保留在仓库中，后续新实例默认直接复制，不依赖临时 Composer 安装。

## 新实例初始化原则

1. 从私有母版代码复制新站。
2. 不带 `runtime/` 运行缓存。
3. 删除 `config/install.lock` 后重新安装。
4. 先确认飞鸟本体前后台正常。
5. 站群插件启用后，必须先完成当前域名/站点配置。
